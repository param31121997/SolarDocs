# SolarDocs Quick Start Guide\n\n## 🚀 Getting Started in 5 Minutes\n\n### Windows Users\n\n1. **Start Backend:**\n   ```batch\n   cd solardocs-backend\n   mvn spring-boot:run\n   ```\n   Wait for message: `Tomcat started on port(s): 8080`\n\n2. **In New Terminal - Start Frontend:**\n   ```batch\n   cd solardocs-frontend\n   npm start\n   ```\n   Browser will open automatically at `http://localhost:4200`\n\n3. **First Steps:**\n   - Go to Customers → Create new customer\n   - Fill in customer details\n   - Go to Documents → Generate → Select Quotation\n   - Add line items and click Generate\n   - PDF will be saved in customer's GeneratedPDF folder\n\n### Mac/Linux Users\n\n1. **Start Backend:**\n   ```bash\n   cd solardocs-backend\n   mvn spring-boot:run\n   ```\n\n2. **Start Frontend:**\n   ```bash\n   cd solardocs-frontend\n   npm start\n   ```\n\n3. **Access Application:**\n   - Open http://localhost:4200 in browser\n\n---\n\n## 📁 What Each Folder Does\n\n```\nsol ardocs-backend/\n├── src/main/java/com/solardocs/\n│   ├── domain/         → Business logic (Customer, Template, etc.)\n│   ├── application/    → Services (DocumentGenerationService, etc.)\n│   ├── infrastructure/ → PDF rendering, JSON storage\n│   ├── api/            → REST controllers\n│   └── config/         → App configuration\n├── pom.xml            → Dependencies\n└── src/main/resources/ → application.properties, templates\n\nsolardocs-frontend/\n├── src/app/\n│   ├── core/          → Services, models, interceptors\n│   ├── features/      → Pages (dashboard, customers, documents)\n│   ├── shared/        → Components (language switcher)\n│   └── app.*          → Root component\n├── src/assets/        → Images, translations\n└── package.json       → npm packages\n\nsolardocs-data/\n├── Config/            → Settings files\n├── Customers/         → Customer folders with documents\n├── Templates/         → HTML templates for PDF\n├── Logs/              → Application logs\n└── Backup/            → Backup files\n```\n\n---\n\n## 🔧 Common Tasks\n\n### Generate a Quotation PDF\n\n1. Dashboard → Customers → Select/Create Customer\n2. Click \"Generate Document\"\n3. Select \"Quotation\" from dropdown\n4. Add line items:\n   - Item: \"5 kW Solar Panel\"\n   - Type: \"Equipment\"\n   - Qty: \"1\"\n   - Unit: \"Set\"\n   - Rate: \"150000\"\n5. Click \"Generate Document\"\n6. PDF saved automatically\n\n### Add a New Customer\n\n1. Dashboard → Customers → New Customer\n2. Fill form:\n   - Name\n   - Mobile\n   - Email\n   - Address\n   - Village\n   - Pincode\n   - State\n3. Click Save\n4. Customer ID auto-generated\n\n### Change Language\n\n1. Click language icon in top-right\n2. Select: English, Hindi, or Marathi\n3. Entire app updates instantly\n\n### View Logs\n\n1. Backend logs: `solardocs-data/Logs/solardocs.log`\n2. Frontend logs: Browser F12 → Console tab\n\n---\n\n## ⚠️ Important Settings\n\n### Backend Data Location\nEdit: `solardocs-backend/src/main/resources/application.properties`\n```properties\n# Change this path to your desired data location\nsolardocs.data-dir=D:\\\\SolarDocsData  # Windows\nsolardocs.data-dir=/home/user/SolarDocsData  # Linux\n```\n\n### API URL (if backend on different machine)\nEdit: `solardocs-frontend/src/environments/environment.ts`\n```typescript\napiBaseUrl: 'http://192.168.1.100:8080/api'\n```\n\n---\n\n## 🐛 Quick Fixes\n\n**\"Cannot connect to server\"**\n- ✓ Backend running on port 8080?\n- ✓ Check firewall settings\n- ✓ Check API URL in environment.ts\n\n**\"Template not found\"**\n- ✓ Templates in solardocs-data/Templates/?\n- ✓ templates-registry.json valid?\n- ✓ Check backend logs\n\n**\"PDF not generating\"**\n- ✓ Required fields filled?\n- ✓ At least one line item added?\n- ✓ Check browser console (F12)\n- ✓ Check backend logs\n\n**\"Data not saving\"**\n- ✓ Folder permissions correct?\n- ✓ Disk space available?\n- ✓ Check solardocs.log\n\n---\n\n## 📊 Dashboard Explained\n\n- **Total Customers:** Count of all customers in system\n- **Pending Documents:** Customers without commission documents\n- **Commissioned (Month):** Systems commissioned this month\n- **Status Breakdown:** Visual breakdown by customer status\n  - Active: Green\n  - Pending: Yellow\n  - Completed: Blue\n  - Inactive: Gray\n\n---\n\n## 🎨 UI Features\n\n✅ **Responsive:** Works on desktop, tablet, mobile\n✅ **Dark Theme Support:** Auto-detects system preference\n✅ **Smooth Animations:** Professional transitions\n✅ **Error Messages:** Clear, helpful feedback\n✅ **Loading Indicators:** Know when app is working\n✅ **Material Design:** Google Material Design components\n\n---\n\n## 📝 Keyboard Shortcuts\n\n- `Ctrl+S` / `Cmd+S` - Save (in forms)\n- `Escape` - Close dialog\n- `Enter` - Submit form\n- `F12` - Open developer tools (debugging)\n\n---\n\n## 🆘 Getting Help\n\n1. **Check Logs:**\n   - Backend: `solardocs-data/Logs/solardocs.log`\n   -
