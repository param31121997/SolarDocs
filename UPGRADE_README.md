packages)\n\n### Backend Setup\n\n1. **Navigate to backend directory:**\n   ```bash\n   cd solardocs-backend\n   ```\n\n2. **Configure data directory (optional):**\n   Edit `src/main/resources/application.properties`:\n   ```properties\n   # Windows\n   solardocs.data-dir=D:\\\\SolarDocsData\n   # Mac/Linux\n   solardocs.data-dir=/home/user/SolarDocsData\n   ```\n\n3. **Build the project:**\n   ```bash\n   mvn clean install\n   ```\n\n4. **Run the backend:**\n   ```bash\n   mvn spring-boot:run\n   ```\n   The server will start on `http://localhost:8080`\n\n### Frontend Setup\n\n1. **Navigate to frontend directory:**\n   ```bash\n   cd solardocs-frontend\n   ```\n\n2. **Install dependencies:**\n   ```bash\n   npm install\n   ```\n\n3. **Update API URL (if backend is on different host):**\n   Edit `src/environments/environment.ts`:\n   ```typescript\n   export const environment = {\n     apiBaseUrl: 'http://your-backend-url:8080/api'\n   };\n   ```\n\n4. **Start development server:**\n   ```bash\n   npm start\n   ```\n   The app will open at `http://localhost:4200`\n\n### Data Directory Structure\n\nEnsure your data directory has the Troubleshooting\n\n### PDF Generation Issues\n\n**Problem:** Template not found\n- **Solution:** Ensure template HTML files are in `SolarDocsData/Templates/` directory\n- Check `templates-registry.json` has correct template definitions\n- Verify file names match exactly (case-sensitive)\n\n**Problem:** PDF rendering fails\n- **Solution:** Check backend logs at `SolarDocsData/Logs/solardocs.log`\n- Verify Thymeleaf template syntax is correct\n- Ensure all required fields are provided in the API request\n\n### Connection Issues\n\n**Problem:** Cannot connect to backend\n- **Solution:** Verify backend is running on port 8080\n- Check API URL in `environment.ts`\n- Ensure no firewall blocking port 8080\n- Check browser console for CORS errors\n\n**Problem:** CORS errors\n- **Solution:** Backend CORS is configured in `com.solardocs.config.CorsConfig`\n- Verify frontend URL matches allowed origins\n\n### Data Issues\n\n**Problem:** Customer data not saving\n- **Solution:** Check `SolarDocsData/Customers/` directory permissions\n- Ensure disk space is available\n- Check `solardocs.log` for specific errors\n\n**Problem:** Missing translations\n- **Solution:** Check `src/assets/i18n/` directory for language JSON files\n- Verify translation keys exist in all language files\n- Check browser console for missing translation warnings\n\n---\n\n## API Endpoints\n\n### Templates\n```\nGET /api/templates\n- Get all active templates\n```\n\n### Document Generation\n```\nPOST /api/customers/{id}/documents/generate\n- Generate a PDF document for a customer\n- Body: { templateCode: string, extraFields: object }\n```\n\n### Customers\n```\nGET /api/customers\n- List all customers\n\nGET /api/customers/{id}\n- Get customer details\n\nPOST /api/customers\n- Create new customer\n\nPUT /api/customers/{id}\n- Update customer\n```\n\n---\n\n## Performance Tips\n\n1. **Backend:** Use SSD storage for data directory\n2.
